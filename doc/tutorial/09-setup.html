Prepare database

The domain properties would go into a different table than the collection. So we will create another table in openDatabase and insert a default row, to not have to handle undefined values. For now there will only be one row in it, but we will already identify it with an id, which is useful for being able to change the collection name. Though the id will only be a hardcoded placeholder id at the top of "data.js" for now.

var id = "placeholderid";

In openDatabase, after the lines for creating the "Collection" table and filling it with the sample values, create and fill a "Domain" table:

tx.executeSql('CREATE TABLE IF NOT EXISTS Domain(id TEXT, name TEXT, ' +
              'domain TEXT, ' +
              'ageContextPrefix TEXT, ageContextSuffix TEXT, ' +
              'borderWidth NUMERIC, ' +
              'textureType NUMERIC, smooth NUMERIC, ' +
              'textureHeight NUMERIC, textureWidth NUMERIC, ' +
              'mainColor TEXT, textureColor TEXT, borderColor TEXT)');
res = tx.executeSql('SELECT * FROM Domain WHERE id = \'' + id + '\'');
if (res.rows.length === 0)
    tx.executeSql('INSERT INTO Domain VALUES(?,?,?,?,?,?,?,?,?,?,?,?,?)',
                  [id, "", "", "", "", 0, 0, 0, 100, 100, "", "", ""]);

The '100' for the texture height and width is just an arbitrary value to make the texture visible if the size is not yet set and the texture is selected. This matches a default initialization of (to - from) / 4 for the sliders, so at a quarter of a slider.

The color is not initialized by default to e.g. black, as this would be handled by defaultColor we added in a previous chapter.

Now we need methods to read and write single values. While this could lead to performance issues unlike just saving once on pressing a button, this should rarely be the case nowadays. We will add two general methods, and then call them from smaller methods for each property.

function setDomainValue(col, value)
{
    var db = openDatabase();
    db.transaction(function(tx) {
        tx.executeSql('UPDATE Domain SET ' + col + ' = \'' + value + '\' ' +
                      'WHERE id = \'' + id + '\'');
    });
}

function getDomainValue(col)
{
    var item = "";
    var db = openDatabase();
    db.transaction(function(tx) {
        var res = tx.executeSql('SELECT ' + col + ' FROM Domain ' +
                                'WHERE id = \'' + id + '\'');
        if (res.rows.length === 0 )
            return;
        item = res.rows.item(0);
    });
    return item;
}

The lot of smaller methods all have the same structure, so I will only show one and the special case for 'smooth'. For the other methods remember returning 100 as default value for texture height and width, and an empty string for the colors.

function setDomainName(name)
{
    setDomainValue("name", name);
}

function getDomainName()
{
    var item = getDomainValue("name");
    return item === "" ? "" : item.name;
}

For 'smooth' we take and return a boolean, but get and write a number:

function setDomainTextureSmooth(isSmooth)
{
    setDomainValue("smooth", isSmooth ? 1 : 0);
}

function getDomainTextureSmooth()
{
    var item = getDomainValue("smooth");
    return item !== "" && item.smooth !== 0;
}

The texture type is set as number using the index of the type. This is not robust and should be done differently for growing project sizes.

For the age context there can be a third method that returns the combined string. While it could just be a getter calling the prefix and suffix methods, let's make it independent from those methods and instead just format parameters. We will want to use the method with input item text in the settings view.

function formatDomainAgeContext(prefix, suffix)
{
    return (prefix + " %1 %2 " + suffix).trim();
}

In sum there should be 12x2 smaller methods. With those we have an interface to get the collection properties from the database and save them.

We can now also make the properties required, so we won't have to specify hardcoded default values in different places. Layout name and domain input

Both the name of a collection and the domain are arbitrary strings, so they would be set through input items. As they should also show the current value from the database, the hint can not be used to explain the input type; at least not if the value is available. So we need a label for the input item. And even with the label the input type might not be clear, so a short explanation with an example should also be shown. Since showing it directly as text would take up quite a lot of space, an info button should be shown next to the input that could e.g. open a dialog with the description and example.

So add a component file LabeledTextField.qml. The needed imports are QtQuick, QtQuick.Layouts and Felgo.

To fill the width of the parent component and have the input item fill the space between label and icon button, the base type will be a RowLayout. It has required properties for the label, the input placeholder and the dialog message, and an optional property for an initial input text. The child items are AppText for the label, the AppTextField itself and an IconButton with IconType.infocircle. The message can be shown inside a Dialog, but it should be added to the settings view directly, as it would otherwise be opened inside the row instead of the full page. It should have properties for the message (string message) and a general title qsTr("Info"). Also add a signal infoClicked to the row layout and call it when the button is clicked.

RowLayout {
    id: labeledTextField
    required property string label
    required property string placeholderText
    property string text
    required property string info
    signal infoClicked()
    spacing: 10
    width: parent.width

    AppText {
        id: txtLabel
        text: labeledTextField.label
        Layout.alignment: Qt.AlignVCenter
    }

    AppTextField {
        id: input
        placeholderText: labeledTextField.placeholderText
        text: labeledTextField.text
        onTextChanged: labeledTextField.text = text
        Layout.alignment: Qt.AlignVCenter
        Layout.fillWidth: true
    }

    IconButton {
        iconType: IconType.infocircle
        Layout.alignment: Qt.AlignVCenter
        Layout.preferredHeight: txtLabel.height
        Layout.preferredWidth: Layout.preferredHeight
        onClicked: infoClicked()
    }
}

Dialog { // in CollectionSettingsView.qml

Get random colors from database

Add an on-clicked handler to the button for randomizing colors. We will write a method in data.js that returns an array of three colors: the main color, texture color and border color.

IconButton {
    id: btnRandomColors
    ...
    onClicked: {
        var colors = Data.getRandomColors();
        colColors.defaultMainColor = colors[0];
        colColors.defaultTextureColor = colors[1];
        colColors.defaultBorderColor = colors[2];
    }
}

Inside the method, define variables for each color. Then start a transaction to read the corresponding columns from the collection table, randomize an index over the length of the results and write the colors into the variables.

function getRandomColors()
{
    var mainColor = "";
    var textureColor = "";
    var borderColor = "";

    var db = openDatabase();
    db.transaction(function(tx) {
        var res = tx.executeSql('SELECT mainColor, textureColor, borderColor ' +
                                'FROM Collection');
        if (res.rows.length === 0)
            return;

        var index = Math.floor(Math.random() * res.rows.length);
        var item = res.rows.item(index);
        mainColor = item.mainColor;
        textureColor = item.textureColor;
        borderColor = item.borderColor;
    });

    return [mainColor, textureColor, borderColor];
}

And that's it, now you can load sample colors from other items in the database.
