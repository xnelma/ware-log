Display Data in List View

In this chapter we will define a list view, using the Felgo component AppListView, and a component for its delegate.

Add basic list view

For starters, we will show the title of each item in a list view. Add the AppListView component inside AppPage. A margin between entries of 10 units would be good, so add the spacing property with value 10, and a top anchor with a top margin of 10.

AppListView {
    spacing: 10
    anchors.top: parent.top
    anchors.topMargin: 10
}

Then add a model and delegate: The model will be a ListModel, which will be filled on Component.onCompleted:

AppListView {
    ...

    model: ListModel {
        id: listModel

        Component.onCompleted: Data.get(listModel);
    }
}

"Data" comes from the import of the javascript file:

import "data.js" as Data

For a basic delegate, add an Item with the parents width (I defined pageMain as id of AppPage) and a text components height plus some margin. As text component, use the Felgo component AppText, aligned to the left with a margin of 10, and centered vertically.

The model data is specified as required property for Item, and then assigned to the text property of the AppText component:

delegate: Item {
    required property string title

    width: pageMain.width;
    height: txtTitle.height + 10

    AppText {
        id: txtTitle

        text: title

        anchors.verticalCenter: parent.verticalCenter
        anchors.left: parent.left
        anchors.leftMargin: 10
    }
}

Building and deploying the app now should show a list of titles.

Add detailed component for delegate

Next create a new qml Document, I will name it CollectionDelegate. This will contain the layout for a delegate. The base component is a Item.

Specify model data

Similar to the required property for the title in the previous section, define required properties for every entry:

required property string title
required property string origin
required property string originColor
required property string type
required property int age
required property string ageUnit
required property int weightTotal
required property int weightLeft
required property string weightUnit
required property int cost
required property int secondaryCost
required property string mainColor
required property string textureColor
required property string borderColor
required property string tagStr
property list<string> tags

The list of tags will be created from tagStr if the tags are not passed as list. While tagStr is also not mandatory, it still needs to be marked as required to be associated with model data in case the delegate is used inside a list view.

Component.onCompleted: {
    if (tags.length == 0)
        tags = Data.tagList(tagStr);
}

There should also be a property ageDescription to give some domain specific context to the meaning of the entry type. I picked it to be:

property string ageDescription: qsTr("collected %1 %2 ago")

General structure of the layout

To have a robust layout that will e.g. match for each item (would be a problem with a grid view that scales cells according to their content), not break as easily with different text lengths (would be a problem if I had a grid with a dynamic width for columns on e.g. the left, and then rightermost columns would disappear out of the right side of the screen), and give an overview over the data (so not just list it), I chose to have a colored rectangle on the left with a fixed width calculated from the parent's width, and a Column for the text on the right side of it.

While entries could have a pictures instead of the colored rectangle to show the characteristics of an item, one base idea for this app was targeting items that are measured in e.g. grams, so are rather homogeneous. Not having to save pictures means using less storage space. The colored rectangle also only serves as a base for further ideas for custom components.

Rectangle {
    id: rectWeight

    height: parent.height
    width: delegate.width / 7

    color: delegate.mainColor
}

Column {
    id: colContent

    anchors.left: rectWeight.right
    anchors.leftMargin: 10

    ...
}

While the height of the rectangle is set according to the parent height, the parent height depends on the content of the text column, with some padding:

Item {
    id: delegate

    height: colContent.height + 2

    ...
}

As a small add-on, it would also be nice to be able to set the text color of the item. There is no background set so it would already be possible to change it from outside the component, but the text color would be fix. So add another property that would per default be set to the theme color.

property color textColor: Theme.colors.textColor

Layout of properties title and origin

The first line of the content column will be title and origin. The origin is colored.

Row {
    spacing: 10
    AppText {
        id: txtTitle

        text: delegate.title
        color: delegate.textColor
        font.bold: true
    }

    AppText {
        id: txtOrigin

        text: delegate.origin
        color: delegate.originColor
    }
}

Layout of the type property

The type will have a dark grey background, the tag list later on will have a lighter grey background with round corners.

Item {
    height: txtType.height + 2
    width: txtType.width + 4

    Rectangle {
        height: parent.height
        width: parent.width
        color: txtType.color
        opacity: 0.2
    }

    AppText {
        id: txtType
        text: delegate.type
        color: delegate.textColor
        anchors.centerIn: parent
    }
}

Layout of the price

The price will be per 10g, or another fix amount. If the secondary and primary price are not equal, the first price will be formatted with a strikeout, and the other price is shown. Make the color of the amount a softer color than the price to make it stand out less. Setting the opacity instead of a color makes switching to a different font color, e.g. for a dark mode, easier.

Row {
    spacing: 2

    AppText {
        text: delegate.cost + "€"
        font.strikeout: delegate.cost !== delegate.secondaryCost
        color: delegate.textColor
    }

    AppText {
        text: delegate.secondaryCost + "€"
        visible: delegate.cost !== delegate.secondaryCost
        color: delegate.textColor
    }

    AppText {
        text: "/10g"
        opacity: 0.7
        color: delegate.textColor
    }
}

Layout of the age property

This will use the age description property we defined earlier, and can be a simple AppText component:

AppText {
    text: delegate.ageDescription.arg(delegate.age)
                                 .arg(delegate.ageUnit)
    color: delegate.textColor
}

Layout for the tag list

Now for the tag list. We will format it as a Repeater in a Row. The model will be the string list for the tags we parsed earlier. It is available as property modelData of type string. Each tag should have a light grey background with round corners, but not half circles, so decrement the radius with 2. The light grey is achieved by using the text color of the text item, and setting an opacity of 0.1. The text is also not black, but a bit lighter, using an opacity of 0.8.

Row {
    Repeater {
        model: delegate.tags

        Item {
            required property string modelData

            height: rectPropBg.height
            width: rectPropBg.width + 3

            AppText {
                id: txtProp
                text: parent.modelData
                opacity: 0.8
                color: delegate.textColor
                anchors.centerIn: rectPropBg
            }

            Rectangle {
                id: rectPropBg
                color: txtProp.color
                opacity: 0.1
                radius: height / 2 - 2
                height: txtProp.height + 4
                width: txtProp.width + 10
            }
        }
    }
}

Use delegate component

Assign the delegate component to the delegate property:

delegate: CollectionDelegate {
    width: pageMain.width
}

Here we will also set its width.

Now if you build and deploy the app, you should see a detailed list view with the sample data added in the first chapter.
