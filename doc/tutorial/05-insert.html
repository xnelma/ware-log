Insert Item

It is now possible to view the database items, and delete items. Inserting items is not implemented yet. For that we will add a floating button and the navigation to another page, write a layout for the page and implement inserting the item into the database.

Add floating button for navigation

In Felgo there is the FloatingActionButton component, which will not require manual positioning but add the button to the bottom right corner by default. Add this button to AppPage in Main.qml.

FloatingActionButton {
    iconType: IconType.plussquareo
    onClicked: console.log("clicked");
    visible: true
}

The property visible: true is necessary, as it is not the default on all platforms.

Navigate to new page

Add another qml file in the qml folder with the title EditItemView. This will be the view we will navigate to for editing an item; in this case editing a new item to insert.

The base item should be an AppPage, and might have a title that would get shown in a toolbar. Put a text item (and imports for QtQuick, QtQuick.Layouts und Felgo) in it.

AppPage {
    title: qsTr("Add New Item")

    AppText {
        text: "navigated to EditItemView"
        anchors.centerIn: parent
    }
}

Navigating to the view could be done via the navigation stack, or for example by showing it inside a dialog, preferably a modal dialog, so a dialog that blocks the UI beneath it. In the Felgo SDK there is the AppModal type. It does not inherit from Dialog but Item, so dialog buttons are not part of its properties and will have to be added to the view.

Add two AppButtons in a Row, with alignment to the right. Remove margins, and add a row spacing, so they will align well with other page content.

AppPage {
    title: qsTr("Add New Item")

    ...

    Row {
        spacing: 10
        anchors.right: parent.right

        AppButton {
            text: qsTr("Cancel")
            horizontalMargin: 0
            verticalMargin: 0
            onClicked: {
                modalAddItem.close()
            }
        }

        AppButton {
            text: qsTr("Add item")
            horizontalMargin: 0
            verticalMargin: 0
            onClicked: {
                modalAddItem.close();
                console.log("Add item");
            }
        }
    } // Row
}

In Main.qml, add AppModal with id modalAddItem as child of AppPage:

AppModal {
    id: modalAddItem
    pushBackContent: root.navigationStack
    fullscreen: false
    modalHeight: modalContent.contentHeight

    EditItemView { id: modalContent }
}

It will not cover the whole screen, so the height needs to be set. Add a property contentHeight to the view, and move the buttons and text (remove the anchor from text!) into a column, with spacing and margins of 10. contentHeight will then be set with the column height, plus some margin.

AppPage {
    title: qsTr("Add New Item")

    property int contentHeight: col.height
                                + 40 // add bottom margin

    Column {
        id: col
        spacing: 10
        width: parent.width
        anchors.top: parent.top
        anchors.left: parent.left
        anchors.right: parent.right
        anchors.margins: 10

        AppText { ... }

        Row { AppButton { ... } ... }
    }
}

Separate the row of buttons and text visually with a MenuSeparator. We now also require the import for QtQuick.Controls. Remove the padding from the separator, too, so it aligns with the buttons.

AppText { ... }

MenuSeparator {
    width: parent.width
    padding: 0
}

Row { AppButton { ... } ... }

Then open the dialog when clicking the floating button:

FloatingActionButton {
    ...
    onClicked: modalAddItem.open()
    ...
}

Add layout for new page

The layout for the item input follows the structure of a list item.

Layout title and origin input

As title and origin will be on the same line in a list item, we could put them on the same line here, too. This might give an impression for word length. We can use a Row for that, and center its items vertically, in case they have different heights. The input width should be the same for both, but not depend on each other, so we add a property inputWidth that computes it by parent width, color button width and the margin.
The color button would be for setting a 'signature color' for origin. Both the origin text (and its placeholder) and the button should show the current color. Choose a placeholder color for now. For AppButton the text margin can get removed by setting a low minimum width.

All in all, for the first line of the column in the dialog, we add the following Row:

Row {
    id: rowTitle
    spacing: 10
    width: parent.width
    property string placeholderOriginColor: "#457" // Dark softer blue

    property int inputWidth:
        (parent.width - btnSetOriginColor.width - 10) / 2;

    AppTextField {
        placeholderText: qsTr("Item title")
        width: rowTitle.inputWidth
        anchors.verticalCenter: parent.verticalCenter
    }

    AppTextField {
        placeholderText: qsTr("Origin")
        color: rowTitle.placeholderOriginColor
        placeholderColor: rowTitle.placeholderOriginColor
        width: rowTitle.inputWidth
        anchors.verticalCenter: parent.verticalCenter

    }

    AppButton {
        id: btnSetOriginColor
        text: "Color"
        backgroundColor: rowTitle.placeholderOriginColor
        minimumWidth: 10
        anchors.verticalCenter: parent.verticalCenter
    }
} // rowTitle

Layout type selection

Next in a list item comes the type. Add an editable ComboBox with some placeholder types.

ComboBox {
    model: ["Type 1", "Type 2"]
    editable: true
}

Layout color selection

Now we add a row for setting the colors for a list item. There will be two colors, so add two buttons. Remove their margins, so they will align well with other ui elements. Also, pick two placeholder colors and set them as the buttons' background colors.

Row {
    id: rowColors
    spacing: 10

    property string placeholderPrimaryColor: "#e3aae3" // light pink
    property string placeholderSecondaryColor: "#a353a3" // darker pink

    AppButton {
        text: "Color 1"
        backgroundColor: rowColors.placeholderPrimaryColor
        horizontalMargin: 0
        verticalMargin: 0
    }

    AppButton {
        text: "Color 2"
        backgroundColor: rowColors.placeholderSecondaryColor
        horizontalMargin: 0
        verticalMargin: 0
    }
} // rowColors

Layout cost, weight and age input

The following three rows will set a number input, and a unit for it. The input width should be the same, so first add a property to the Column, after rowColors:

property int numFieldWidth: 100

Each of the inputs will be inside a Row, together with a ComboBox and maybe a label, with all elements centered vertically in the Row.

Row {
    spacing: 10

    AppTextField {
        placeholderText: qsTr("Cost")
        inputMethodHints: Qt.ImhDigitsOnly
        width: col.numFieldWidth
        anchors.verticalCenter: parent.verticalCenter
    }

    AppText {
        text: "€ / "
        anchors.verticalCenter: parent.verticalCenter
    }

    ComboBox {
        model: ["10g", "100g", "1kg", "10ml", "100ml", "1l"]
        anchors.verticalCenter: parent.verticalCenter
    }
} // Row

Row {
    spacing: 10

    AppTextField {
        placeholderText: qsTr("Weight")
        inputMethodHints: Qt.ImhDigitsOnly
        width: col.numFieldWidth
        anchors.verticalCenter: parent.verticalCenter
    }

    ComboBox {
        id: comboWeightUnit
        model: ["g", "kg", "l", "ml"]
        anchors.verticalCenter: parent.verticalCenter
    }
} // Row

Row {
    spacing: 10

    AppTextField {
        placeholderText: qsTr("Age")
        inputMethodHints: Qt.ImhDigitsOnly
        width: col.numFieldWidth
        anchors.verticalCenter: parent.verticalCenter
    }

    ComboBox {
        id: comboAgeUnit
        model: ["months", "years"]
        anchors.verticalCenter: parent.verticalCenter
    }
} // Row

Layout tags selection

For the tags, we use a Repeater inside a Flow item, with a checkable AppButton as delegate. The Flow item will distribute the items over multiple lines as needed. The button is styled to look similar to the tags in a list item: the background color is a grey tone (darker, if a tag is selected), the shadow is removed, the corners are rounded, and the text margin is removed to wrap its text. The model will be set to placeholder tags for now.

The last item in the Repeater is a text field to add a new tag.

Flow {
    width: parent.width
    spacing: 10

    Repeater {
        anchors.horizontalCenter: parent.horizontalCenter
        model: ["Tag 1", "Tag 2", "Tag 3"]
        delegate: AppButton {
            text: modelData
            checkable: true
            backgroundColor: checked? "#737373" : "#c3c3c3"
            dropShadow: false
            radius: height/2 - 5
            minimumWidth: 10
            horizontalMargin: 0
            verticalMargin: 0
        }
    } // Repeater

    AppTextField {
        placeholderText: qsTr("+ New tag")
    }
} // Flow

The next item in the column should be the separator we added in a previous section.
