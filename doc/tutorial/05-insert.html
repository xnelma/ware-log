Insert Item

It is now possible to view the database items, and delete items. Inserting items is not implemented yet. For that we will add a floating button and the navigation to another page, write a layout for the page and implement inserting the item into the database.

Add floating button for navigation

In Felgo there is the FloatingActionButton component, which will not require manual positioning but add the button to the bottom right corner by default. Add this button to AppPage in Main.qml.

FloatingActionButton {
    iconType: IconType.plussquareo
    onClicked: console.log("clicked");
    visible: true
}

The property visible: true is necessary, as it is not the default on all platforms.

Navigate to new page

Add another qml file in the qml folder with the title EditItemView. This will be the view we will navigate to for editing an item; in this case editing a new item to insert.

The base item should be an AppPage, and might have a title that would get shown in a toolbar. Put a text item (and imports for QtQuick, QtQuick.Layouts und Felgo) in it.

AppPage {
    title: qsTr("Add New Item")

    AppText {
        text: "navigated to EditItemView"
        anchors.centerIn: parent
    }
}

Navigating to the view could be done via the navigation stack, or for example by showing it inside a dialog, preferably a modal dialog, so a dialog that blocks the UI beneath it. In the Felgo SDK there is the AppModal type. It does not inherit from Dialog but Item, so dialog buttons are not part of its properties and will have to be added to the view.

Add two AppButtons in a Row, with alignment to the right. Remove margins, and add a row spacing, so they will align well with other page content.

AppPage {
    title: qsTr("Add New Item")

    ...

    Row {
        spacing: 10
        anchors.right: parent.right

        AppButton {
            text: qsTr("Cancel")
            horizontalMargin: 0
            verticalMargin: 0
            onClicked: {
                modalAddItem.close()
            }
        }

        AppButton {
            text: qsTr("Add item")
            horizontalMargin: 0
            verticalMargin: 0
            onClicked: {
                modalAddItem.close();
                console.log("Add item");
            }
        }
    } // Row
}

In Main.qml, add AppModal with id modalAddItem as child of AppPage:

AppModal {
    id: modalAddItem
    pushBackContent: root.navigationStack
    fullscreen: false
    modalHeight: modalContent.contentHeight

    EditItemView { id: modalContent }
}

It will not cover the whole screen, so the height needs to be set. Add a property contentHeight to the view, and move the buttons and text (remove the anchor from text!) into a column, with spacing and margins of 10. contentHeight will then be set with the column height, plus some margin.

AppPage {
    title: qsTr("Add New Item")

    property int contentHeight: col.height
                                + 40 // add bottom margin

    Column {
        id: col
        spacing: 10
        width: parent.width
        anchors.top: parent.top
        anchors.left: parent.left
        anchors.right: parent.right
        anchors.margins: 10

        AppText { ... }

        Row { AppButton { ... } ... }
    }
}

Separate the row of buttons and text visually with a MenuSeparator. We now also require the import for QtQuick.Controls. Remove the padding from the separator, too, so it aligns with the buttons.

AppText { ... }

MenuSeparator {
    width: parent.width
    padding: 0
}

Row { AppButton { ... } ... }

Then open the dialog when clicking the floating button:

FloatingActionButton {
    ...
    onClicked: modalAddItem.open()
    ...
}

Get tags and types model from database

First, prepare a list of types and tags to choose from. This would be types and tags already set for another item in the database; though it will also be possible to add new values.

Import the database javascript file for this qml component (import "data.js" as Data).

Then add two methods to it, getAllTags and getAllTypes. They can then be used for the model properties with model: Data.getAllTypes() and model: Data.getAllTags().

The implementation will select the respective columns and add those values to an array, that are not in it yet. I used straight forward loops and an empty inital array, but there are probably better ways to write this.

function getAllTags()
{
    const allTags = [];
    var db = openDatabase();
    db.transaction(function(tx) {
        var res = tx.executeSql('SELECT tags FROM Collection');
        for (var i = 0; i < res.rows.length; i++) {
            var itemTags = tagList(res.rows.item(i).tags);
            for (var j = 0; j < itemTags.length; j++) {
                if (!allTags.includes(itemTags[j]))
                    allTags.push(itemTags[j]);
            }
        }
    });
    return allTags;
}

function getAllTypes()
{
    const allTypes = [];
    var db = openDatabase();
    db.transaction(function(tx) {
        var res = tx.executeSql('SELECT type FROM Collection');
        for (var i = 0; i < res.rows.length; i++) {
            var itemType = res.rows.item(i).type;
            if (!allTypes.includes(itemType))
                allTypes.push(itemType);
        }
    });
    return allTypes;
}

Layout title and origin input

As title and origin will be on the same line in a list item, we could put them on the same line here, too. This might give an impression for word length. We can use a Row for that, and center its items vertically, in case they have different heights. The input width should be the same for both, but not depend on each other, so we add a property inputWidth that computes it by parent width, color button width and the margin.
The color button would be for setting a 'signature color' for origin. Both the origin text (and its placeholder) and the button should show the current color. The default color will be black. For AppButton the text margin can get removed by setting a low minimum width.

All in all, for the first line of the column in the dialog, we add the following Row:

Row {

        (parent.width - btnSetOriginColor.width - 10) / 2;

        placeholderText: qsTr("Origin")

        placeholderColor: rowTitle.originColor

    AppButton {

Layout tags selection

For the tags, we use a Repeater inside a Flow item, with a checkable AppButton as delegate. The Flow item will distribute the items over multiple lines as needed. For the model, we can now use the method for retrieving the types we wrote in a previous section.

Flow {
    width: parent.width
    spacing: 5

    ...

    Repeater {
        model: Data.getAllTags()
        delegate: ...
    } // Repeater
}

The AppButton is styled to look similar to the tags in a list item: the background color is a grey tone (darker, if a tag is selected), the shadow is removed, the corners are rounded, and the minimum width is set to a small value to match the width of its text.
There will also be an icon to make it clearer that the tags (buttons) need to be clicked to select them.

component TagButton: AppButton {
    iconRight: checked? IconType.checksquareo : IconType.squareo
    checkable: true
    backgroundColor: checked? "#737373" : "#c3c3c3"
    dropShadow: false
    radius: height/2 - 2
    minimumWidth: 10
    horizontalMargin: 0
    verticalMargin: 0
}

The last item in the Repeater is a text field and button to add a new tag. For simplicity, the new tags are added to a second repeater with by default enabled tags (checked buttons) as delegate. This is also why we defined an inline component for the AppButton: to not have to rewrite the delegate for both repeaters.

But, a tag should not be added if it is already in a model; either of the database tags or the new tags. Then the corresponding tag should be enabled. For that, define an inline component for the Repeaters, and write a method that looks for the tag in its model and enables the button (if it is not enabled yet).

component TagRepeater : Repeater {
    id: tagRepeater
    function enableMatchingTag(tag) {
        // Returns false if no matching tag is found.
        for (var i = 0; i < tagRepeater.model.length; i++)
            if (tagRepeater.model[i].toLowerCase()
                === tag.toLowerCase()) {
                // Check if not checked yet.
                if (tagRepeater.itemAt(i).checked !== true)
                    tagRepeater.itemAt(i).checked = true;
                return true;
            }
        return false;
    }
}

The tag would not be added anywhere to the database yet. Only when the item is added to the database with the new tag enabled, it will be part of the tag list of the next item edit view; so when editing the next new item.

Flow {
    width: parent.width
    spacing: 5

    component TagButton : AppButton { ... }
    component TagRepeater : Repeater { ... }

    TagRepeater {
        model: Data.getAllTags()
        delegate: TagButton {
            text: modelData
        }
    } // Repeater

    TagRepeater {
        id: repeaterNewTags

        property list<string> newTags;

        model: newTags
        delegate: TagButton {
            text: modelData
            checked: true
        }
    } // Repeater

    AppTextField {
        id: inputNewTag
        placeholderText: qsTr("New tag")
    }

    AppButton {
        text: "+"
        onClicked: {
            var newTag = inputNewTag.text;
            if (newTag === "")
                return;
            if (repeaterTags.enableMatchingTag(newTag)
                || repeaterNewTags.enableMatchingTag(newTag)) {
                inputNewTag.clear();
                return;
            }
            repeaterNewTags.newTags.push(newTag);
            inputNewTag.clear();
        }
        radius: height / 2
        minimumWidth: 0
        horizontalMargin: 0
        verticalMargin: 0
    }
} // Flow

The next item in the column should be the separator we added in a previous section.

Add color dialogs

We still need to show e.g. a dialog when clicking the color buttons. There is the QML Type ColorDialog we can use. Add it to the bottom of the file, inside AppPage, after the Column and give it an id. The type requires the import for QtQuick.Dialogs.

ColorDialog {
    id: dialogColor
}

This dialog should be opened when one of the three buttons are clicked. It already has a full layout, including 'Cancel' and 'OK' buttons that close the dialog. We will have to set the initial color of the dialog depending on the clicked button, and get the selected color when the 'OK' button is clicked. But first, because it will be the same behavior for three buttons, just with different colors, we will add a component ColorButton to the bottom of the file, above ColorDialog.

component ColorButton : AppButton {
    property string colorName
    backgroundColor: colorName
}

Replace the three buttons with this component, and set the color property instead of their background color.
The color type of the dialog has the QML type color, but we will keep using string because this is the only type we will use it as for now, namely for setting color properties and saving it to the database.

Now for opening the dialog and getting the result. Define a Connection for onAccepted, so we can set the colorName property.

Connections {
    id: connectDialogColor
    target: dialogColor
    enabled: false
    function onAccepted() {
        colorName = dialogColor.selectedColor;
        connectDialogColor.enabled = false;
    }
}

The connection also needs to be disabled again right away, so it does not get the signal when one of the other buttons is clicked, too.

In the onClicked handler, set the color of the dialog, enable the connection and open the dialog:

onClicked: {
    dialogColor.selectedColor = colorName;
    connectDialogColor.enabled = true;
    dialogColor.open();
}

The overall structure of the inline component should now be as follows:

component ColorButton : AppButton {
    property string colorName
    backgroundColor: colorName
    onClicked: { ... }

    Connections { }
}

The three buttons also need to handle the color name change, as the Property Bindings to it are one sided:

ColorButton {
    ...
    colorName: rowTitle.originColor
    onColorNameChanged: rowTitle.originColor = colorName
    // or rowColors.primaryColor or rowColors.secondaryColor
    ...
}

Now the colors can be changed through a color dialog.

Check input fields

If a field is empty, it should be highlighted until edited. An item with empty fields should not be created. For that, add another component CheckedTextField with a method to check and set a red color on the field. It will also have a connection that is enabled until the field is edited: it resets the highlight color to the previous color.
If it is already enabled, just return false to avoid resetting properties, especially the cached color that would then be overriden with the highlight color.

 component CheckedTextField : AppTextField {
    id: inputChecked
    property color prevPlaceholderColor

    function ok() {
        if (inputChecked.text !== "")
            return true;
        if (resetPlaceholderColor.enabled)
            return false;
        inputChecked.prevPlaceholderColor = inputChecked.placeholderColor;
        inputChecked.placeholderColor = "red";
        resetPlaceholderColor.enabled = true;
        return false;
    }

    Connections {
        id: resetPlaceholderColor
        target: inputChecked
        enabled: false
        function onFocusChanged() {
            inputChecked.placeholderColor = inputChecked.prevPlaceholderColor
            resetPlaceholderColor.enabled = false;
        }
    }
}

Then replace the AppTextField for title, origin, cost, weight and age with CheckedTextField and add ids to the fields.

With the ids, add and call a method fieldsAreFilled to the 'Add item' button.

function fieldsAreFilled() {
    var titleOk = inputTitle.ok();
    var originOk = inputOrigin.ok();
    var costOk = inputCost.ok();
    var weightOk = inputWeight.ok();
    var ageOk = inputAge.ok();
    return titleOk && originOk && costOk && weightOk && ageOk;
}

As the type selection is editable, it can also have an empty field that is not checked for yet. We do something similar as for the text edit fields, but as there is not a straight forward property to set for colors on the combo box, we just show an exclamation mark next to it. For that, move the ComboBox into a Row, add an AppIcon component and center it vertically in the row.

Row {
    spacing: 10

    ComboBox {
        id: comboTypes
        model: Data.getAllTypes();
        editable: true

        function ok() {
            if (comboTypes.editText !== "")
                return true;
            if (txtComboHighlight.visible)
                return false;
            txtComboHighlight.visible = true;
            resetHighlight.enabled = true;
        }

        Connections {
            id: resetHighlight
            target: comboTypes
            enabled: false
            function onEditTextChanged() {
                txtComboHighlight.visible = false;
                resetHighlight.enabled = false;
            }
        }
    }

    AppIcon {
        id: txtComboHighlight
        iconType: IconType.exclamation
        color: "red"
        visible: false
        anchors.verticalCenter: parent.verticalCenter
    }
}

With that check, the fieldsAreFilled method is complete: we would only save an item, if no fields are empty.
