Delete Item

This chapter shows the steps for deleting an item from the list.

Add the delete button

The Felgo SDK provides a component SwipeOptionsContainer, that makes e.g. a list item swipable, and shows a button left or right on swipe.

We will wrap the base component Item in CollectionDelegate.qml in a SwipeOptionsContainer, but to be able to also use the layout for a list item without the swipe buttons, it should be a separate component that includes CollectionDelegate. Create a file SwipableCollectionDelegate.qml with the same properties for the list model as CollectionDelegate, that has the swipe container as root item, and the delegate as child item, with its properties bound to the delegate item.

SwipeOptionsContainer {
    id: swipeOptionsContainer

    height: content.height

    required property string title
    required property string origin
    ...
    property list<string> tags

    property string ageDescription: qsTr("collected %1 %2 ago")

    Component.onCompleted: {
        tags = Data.tagList(tagStr);
    }

    CollectionDelegate {
        id: content

        title: swipeOptionsContainer.title
        origin: swipeOptionsContainer.origin
        originColor: swipeOptionsContainer.originColor
        type: swipeOptionsContainer.type
        age: swipeOptionsContainer.age
        ageUnit: swipeOptionsContainer.ageUnit
        weightTotal: swipeOptionsContainer.weightTotal
        weightLeft: swipeOptionsContainer.weightLeft
        weightUnit: swipeOptionsContainer.weightUnit
        cost: swipeOptionsContainer.cost
        secondaryCost: swipeOptionsContainer.secondaryCost
        mainColor: swipeOptionsContainer.mainColor
        textureColor: swipeOptionsContainer.textureColor
        borderColor: swipeOptionsContainer.borderColor
        tagStr: swipeOptionsContainer.tagStr
        tags: swipeOptionsContainer.tags
    }
}

Use the SwipableCollectionDelegate instead of the CollectionDelegate in the list view in Main.qml.

Then assign a SwipeButton to the rightOption property, with a remove icon.

rightOption: SwipeButton {
    iconType: IconType.remove
    height: parent.height
    onClicked: {
        console.log("delete button clicked");
    }
}

At this point, the list data should get displayed as before, but on swiping to the left on a list item, a delete button is shown.

Write delete methods

With the swipe button we will not have access to the index, but to the item title. In case of deleting from the database, the following query would delete the item with a matching title:

function deleteItem(title)
{
    var db = openDatabase();
    db.transaction(function(tx) {
        tx.executeSql('DELETE FROM Collection WHERE title = \'' + title + '\'');
    });
}

As this would not reload the list model, we define a method on it to delete an item given a title:

model: ListModel {
    id: listModel

    Component.onCompleted: Data.get(listModel);

    function deleteItem(title) {
        for (var i = 0; i < listModel.count; i++)
            if (listModel.get(i).title === title)
                listModel.remove(i);
    }
}

Deleting items by title, especially from the database, also has the advantage of being independent from a different order of items in the view and the database. This might for example happen if items get sorted in the frontend. For large amounts of data, which are not expected for this app, the worse runtime for the search as opposed direct indexing might be noticeable.

Now, on swiping left and clicking the delete button, items will be deleted from the database, and the list view.
